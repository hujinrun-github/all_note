Notion 同步配置说明

FlowSpace 的 Notion 同步默认只把带有指定同步标签的 FlowSpace 笔记推送到 Notion；从 Notion 拉回 FlowSpace 需要在同步面板里手动触发。

Data Source ID（数据源 ID）是什么

Data Source ID 是你希望 FlowSpace 写入的 Notion 数据源标识。它通常来自 Notion 数据源或数据库页面的 URL，不是页面标题，也不是集成 Token。

配置时，先在 Notion 中准备一个用于存放 FlowSpace 笔记的数据源，并把它 share 给你的 Notion integration。FlowSpace 会用这个 ID 确认笔记应该同步到 哪个 Notion 数据源里。

Token environment variable（令牌环境变量）是什么

这个输入框填写的是“后端环境变量的名字”，不是 Notion Token 的原文。 默认值是 FLOWSPACE_NOTION_TOKEN。

实际 Token 需要放在启动后端服务的环境变量里。同步时，后端会按这里填写的 变量名读取真实 Token。这样 SQLite 里只保存变量名，不保存敏感令牌。

推荐配置步骤

  1. 在 Notion 创建或选择一个用于 FlowSpace 笔记同步的数据源。
  2. 创建 Notion integration，并把该数据源 share 给这个 integration。
  3. 在启动后端前设置环境变量，例如 FLOWSPACE_NOTION_TOKEN=secret_xxx。
  4. 在 FlowSpace 同步设置中填写 Data Source ID 和环境变量名。
  5. 如果只想同步部分笔记，在同步标签里填写标签名，并给对应笔记打标签。

如果连接测试失败，优先检查三件事：Data Source ID 是否来自正确数据源、 integration 是否已被分享访问该数据源、后端服务启动时是否带上了正确的 Token 环境变量。
